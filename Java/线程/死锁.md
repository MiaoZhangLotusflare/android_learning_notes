# 死锁

　　死锁是这样一种情形：多个线程同时阻塞，它们中的一个或者全部都在等待某个资源被释放，由于线程被无限期地阻塞，因此程序不可能正常终止。

## java 死锁产生的四个必要条件

1. 互斥使用，即当资源被一个线程使用（占用）时，别的线程不能被使用。
2. 不可抢占，资源请求者不能强制从资源占用者手中夺取资源，资源只能由资源占用者主动释放。
3. 请求和保持，即当资源请求者在请求其他的资源的同时保持对原有资源的占用。
4. 循环等待，即存在一个等待队列，P1 占用 P2 的资源，P2 占用 P3 的资源，P3 占用 P1 的资源。这样就形成了一个等待环路。

　　当上述四个条件都成立的时候，便形成死锁。当然，死锁的情况下如果打破上述任何一个条件，便可让死锁消失。

　　解决死锁问题的方法是：一种是用 synchronzied，一种是用 Lock 显式锁实现。

## 死锁代码

```java
public class LockTest {
    public static Integer obj1 = 0;
    public static Character obj2 = 'a';

    public static void main(String[] args) {
        LockA la = new LockA();
        new Thread(la).start();
        LockB lb = new LockB();
        new Thread(lb).start();
    }
}

class LockA implements Runnable {


    @Override
    public void run() {
        try {


            System.out.println(new Date().toString() + " LockA 开始执行");
            while (true) {
                synchronized (LockTest.obj1) {
                    LockTest.obj1++;
                    System.out.println(new Date().toString() + " LockA 锁住 obj1 LockTest.obj1:" + LockTest.obj1);
                    Thread.sleep(3000); // 此处等到是给 B 能锁住机会
                    synchronized (LockTest.obj2) {
                        LockTest.obj2 = (char) (LockTest.obj2 + 1);
                        System.out.println(new Date().toString() + " LockA 锁住 obj2 LockTest.obj2:" + LockTest.obj2);
                        Thread.sleep(60 * 1000); // 为测试，占用了就不放
                    }
                }

            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

class LockB implements Runnable {


    @Override
    public void run() {
        try {
            System.out.println(new Date().toString() + " LockB 开始执行");
            while (true) {
                synchronized (LockTest.obj2) {
                    LockTest.obj2 = (char) (LockTest.obj2 + 1);
                    System.out.println(new Date().toString() + " LockB 锁住 obj2 LockTest.obj2:" + LockTest.obj2);
                    Thread.sleep(3000); // 此处等到是给 A 能锁住机会
                    synchronized (LockTest.obj1) {
                        LockTest.obj1++;
                        System.out.println(new Date().toString() + " LockA 锁住 obj1 LockTest.obj1:" + LockTest.obj1);
                        Thread.sleep(60 * 1000); // 为测试，占用了就不放
                    }
                }

            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
```

　　输出结果：

```java
Tue Jun 09 18:57:25 CST 2020 LockB 开始执行
Tue Jun 09 18:57:25 CST 2020 LockA 开始执行
Tue Jun 09 18:57:25 CST 2020 LockB 锁住 obj2 LockTest.obj2:b
Tue Jun 09 18:57:25 CST 2020 LockA 锁住 obj1 LockTest.obj1:1
```

　　此时死锁产生。

## 解决死锁

　　为了解决上面的死锁问题，不适用显式的去锁，用信号量去控制。

　　信号量可以控制资源能被多少线程访问，这里指定只能被一个线程访问，就做到了类似锁住。而信号量可以指定去获取的超时时间，可以根绝这个超时时间，去做一个额外处理。

　　对于无法成功获取的情况，一般就是重复尝试，或指定尝试的次数，也可以马上退出。

```java
public class UnLockTest {
    public static String obj1 = "obj1";
    public static int num1 = 0;
    public static final Semaphore a1 = new Semaphore(1);
    public static String obj2 = "obj2";
    public static char num2 = 'a';
    public static final Semaphore a2 = new Semaphore(1);

    public static void main(String[] args) {
        LockAa la = new LockAa();
        new Thread(la).start();
        LockBb lb = new LockBb();
        new Thread(lb).start();
    }


}

class LockAa implements Runnable {
    @Override
    public void run() {
        try {
            System.out.println(new Date().toString() + " LockA 开始执行");
            while (true) {
                if (UnLockTest.a1.tryAcquire(1, TimeUnit.SECONDS)) {
                    UnLockTest.num1++;
                    System.out.println(new Date().toString() + " LockA 锁住 obj1 UnLockTest.num1：" + UnLockTest.num1);
                    if (UnLockTest.a2.tryAcquire(1, TimeUnit.SECONDS)) {
                        UnLockTest.num2 = (char) (UnLockTest.num2 + 1);
                        System.out.println(new Date().toString() + " LockA 锁住 obj2 UnLockTest.num2：" + UnLockTest.num2);
                        Thread.sleep(60 * 1000);
                    } else {
                        System.out.println(new Date().toString() + " LockA 锁 obj2 失败 ");
                    }
                } else {
                    System.out.println(new Date().toString() + "LockA 锁 obj1 失败");
                }
                UnLockTest.a1.release(); // 释放
                UnLockTest.a2.release();
                System.out.println(new Date().toString() + " LockA 释放 obj1 和 obj2");
                Thread.sleep(1000); // 马上进行尝试，现实情况下 do something 是不确定的
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

class LockBb implements Runnable {
    @Override
    public void run() {
        try {
            System.out.println(new Date().toString() + " LockB 开始执行");
            while (true) {
                if (UnLockTest.a2.tryAcquire(1, TimeUnit.SECONDS)) {
                    UnLockTest.num2 = (char) (UnLockTest.num2 + 1);
                    System.out.println(new Date().toString() + " LockB 锁住 obj2  UnLockTest.num2：" + UnLockTest.num2);
                    if (UnLockTest.a1.tryAcquire(1, TimeUnit.SECONDS)) {
                        UnLockTest.num1++;
                        System.out.println(new Date().toString() + " LockB 锁住 obj1 UnLockTest.num1：" + UnLockTest.num1);
                        Thread.sleep(60 * 1000);
                    } else {
                        System.out.println(new Date().toString() + " LockB 锁 obj1 失败 ");
                    }
                } else {
                    System.out.println(new Date().toString() + "LockB 锁 obj2 失败");
                }
                UnLockTest.a1.release(); // 释放
                UnLockTest.a2.release();
                System.out.println(new Date().toString() + " LockB 释放 obj1 和 obj2");
                Thread.sleep(10 * 1000); // 这里知识为了眼视，所以 tryAcquire 值用 1 秒，而却 B 要给 A 让出能执行的时间，否则两个永远是死锁
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

```

　　输出结果为：

```java
Tue Jun 09 19:02:15 CST 2020 LockB 开始执行
Tue Jun 09 19:02:15 CST 2020 LockA 开始执行
Tue Jun 09 19:02:15 CST 2020 LockA 锁住 obj1 UnLockTest.num1：1
Tue Jun 09 19:02:15 CST 2020 LockB 锁住 obj2  UnLockTest.num2：b
Tue Jun 09 19:02:16 CST 2020 LockA 锁 obj2 失败 
Tue Jun 09 19:02:16 CST 2020 LockB 锁 obj1 失败 
Tue Jun 09 19:02:16 CST 2020 LockA 释放 obj1 和 obj2
Tue Jun 09 19:02:16 CST 2020 LockB 释放 obj1 和 obj2
Tue Jun 09 19:02:17 CST 2020 LockA 锁住 obj1 UnLockTest.num1：2
Tue Jun 09 19:02:17 CST 2020 LockA 锁住 obj2 UnLockTest.num2：c
```
